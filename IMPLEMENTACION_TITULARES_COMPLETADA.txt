IV. FLUJOS REFACTORIZADOS
================================================================================

ANTES (Arquitectura Antigua):
-------------------------------------------------
Admin → Formulario Certificado → Selecciona User → Genera certificado
Problema: Requería crear usuario previo

AHORA (Nueva Arquitectura):
-------------------------------------------------
Admin → Formulario Certificado → Ingresa DNI → Sistema busca/crea titular → Genera certificado
Beneficio: Puede certificar sin crear usuario del sistema

FLUJO DETALLADO:
1. Admin ingresa DNI en formulario
2. Sistema busca titular por DNI (método: buscarPorDni())
3. Si existe:
   - Reutiliza titular existente
   - Muestra información (nombre, certificados previos)
4. Si NO existe:
   - Admin ingresa nombres y apellidos
   - Sistema crea nuevo titular
   - Vincula certificado al nuevo titular
5. Certificado se guarda con titular_id
6. Usuario puede crearse DESPUÉS y heredar certificados

X. NOTAS IMPORTANTES
================================================================================

⚠️  COMPATIBILIDAD HACIA ATRÁS:
   - user_id_legacy mantiene datos históricos
   - No se eliminan físicamente datos antiguos
   - Posible rollback si es necesario (aunque no recomendado)

⚠️  DATOS INMUTABLES:
   - Los snapshots en certificados son inmutables
   - Usuario no puede cambiar nombres/apellidos del titular
   - Solo admin puede corregir errores de tipeo en titulares

⚠️  RESTRICCIONES DE DELETE:
   - Titular con certificados → No se puede eliminar (RESTRICT)
   - Titular con usuario → No se puede eliminar (validación en controller)
   - Certificado anulado → Se cambia estado, no se elimina

✅ VENTAJAS IMPLEMENTADAS:
   - Certificar sin crear usuarios (objetivo principal logrado)
   - Usuarios heredan certificados al vincularse
   - Verificación pública robusta e independiente
   - Datos certificados inmutables
   - Escalable para grandes volúmenes
