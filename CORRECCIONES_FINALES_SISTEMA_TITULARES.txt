IX. FLUJOS COMPLETOS IMPLEMENTADOS
================================================================================

### FLUJO 1: Generar Certificado con Nuevo Titular ✅
-------------------------------------------------
1. Admin accede a /certificados/generar
2. Ingresa DNI del estudiante
3. Sistema busca titular (AJAX)
4. No encuentra → Muestra alerta amarilla "Nuevo titular"
5. Admin completa nombres y apellidos manualmente
6. Completa datos del certificado
7. Guarda → Sistema crea titular + certificado vinculado
8. Mensaje: "Certificado generado exitosamente"

### FLUJO 2: Generar Certificado con Titular Existente ✅
-------------------------------------------------
1. Admin accede a /certificados/generar
2. Ingresa DNI del estudiante
3. Sistema busca titular (AJAX)
4. Encuentra titular → Alerta verde con info
5. Campos de nombres se bloquean (readonly)
6. Muestra: "Este titular tiene X certificados previos"
7. Admin completa datos del certificado
8. Guarda → Certificado vinculado al titular existente
9. Certificados acumulativos para ese DNI

### FLUJO 3: Crear Usuario Estudiante con Titular Existente ✅
-------------------------------------------------
1. Admin accede a /users/add
2. Selecciona rol "Estudiante"
3. DNI se vuelve obligatorio (marca roja *)
4. Ingresa DNI
5. Sistema verifica titular (AJAX)
6. Encuentra titular sin usuario → Alerta verde
7. Muestra: "Heredará X certificados"
8. Admin completa username, password
9. Guarda → Usuario vinculado a titular
10. Usuario puede ver certificados previos en "Mis Certificados"

### FLUJO 4: Prevenir Usuario Duplicado ✅
-------------------------------------------------
1. Admin accede a /users/add
2. Selecciona rol "Estudiante"
3. Ingresa DNI ya vinculado
4. Sistema verifica titular (AJAX)
5. Encuentra titular CON usuario → Alerta roja
6. Mensaje: "DNI ya vinculado a usuario: username"
7. Botón Guardar DESHABILITADO
8. Admin debe usar otro DNI o editar usuario existente

### FLUJO 5: Editar Usuario Protegiendo DNI ✅
-------------------------------------------------
1. Estudiante accede a su perfil → edit
2. Ve formulario de edición
3. Campo DNI está bloqueado (readonly, fondo gris)
4. Alerta: "DNI protegido, contacte al administrador"
5. Puede editar: username, email, password
6. NO puede cambiar DNI (protección de identidad)
7. Admin SÍ puede cambiar DNI con advertencia

### FLUJO 6: Ver Certificados Heredados ✅
-------------------------------------------------
1. Se generan 3 certificados para DNI 12345678 (sin usuario)
2. Luego se crea usuario con DNI 12345678
3. Sistema vincula automáticamente usuario ↔ titular
4. Estudiante accede a /certificados/misCertificados
5. Sistema busca por titular_id (no por user_id)
6. Estudiante ve los 3 certificados previos + nuevos
7. Herencia completa de certificados

X. VALIDACIONES Y SEGURIDAD
================================================================================

### Validaciones de Base de Datos ✅
-------------------------------------------------
1. titular_id en users → UNIQUE
   - Un titular solo puede tener un usuario
   - FK con ON DELETE RESTRICT

2. titular_id en certificados → NOT NULL
   - Todo certificado debe tener titular
   - FK con ON DELETE RESTRICT

3. dni en titulares → UNIQUE
   - No puede haber DNI duplicados

### Validaciones de Controlador ✅
-------------------------------------------------
1. UsersController::add()
   - Verifica titular existente
   - Valida que no esté vinculado
   - Solo estudiantes requieren DNI

2. UsersController::edit()
   - Protege DNI si está vinculado
   - Solo admin puede modificar DNI vinculado

3. TitularesController
   - requiereAdministrador() en todos los métodos
   - Validación de DNI único
   - No permite eliminar si tiene certificados

### Validaciones de Vista ✅
-------------------------------------------------
1. JavaScript en add.php
   - Timeout de 500ms para evitar múltiples requests
   - Validación de longitud mínima DNI (3 caracteres)
   - Deshabilitación de botón si hay error

2. Formulario generar.php
   - Validación de campos obligatorios
   - Verificación de DNI antes de enviar
   - Protección de campos autocompletados
